Fintech Product Analytics — Churn & A/B Testing

Analysis & ML

A product-analytics project looking at which behaviours line up with an inactivity-based churn label, and whether a simulated onboarding change shifts 30-day transaction activity.

Approach

I generated 8,000 users and 60,000 transactions, ran the SQL analysis in SQLite, trained a Random Forest to classify the churn label, and ran a simulated A/B test with a Welch t-test, a 95% confidence interval, Cohen's d, and a sample-ratio-mismatch check.

Healthcare Claims Analytics

Analysis & ML

A claims-style project on synthetic data, looking at how cost and admissions vary by diagnosis, region and patient risk tier, and whether a model can pick out the admission flag.

Approach

I generated 5,000 patients, 200 providers and 25,000 claims, ran seven SQL queries in SQLite, pulled together cost and utilisation summaries, and trained a Random Forest to classify admissions using a patient-level train/test split, with a logistic-regression baseline for comparison.

Neobank Analytics Engineering — ETL & Metrics Layer

Data Engineering

The layer underneath analysis: turning messy raw source data — mixed date formats, inconsistent country strings, duplicates, multi-currency amounts — into a clean, tested, star-schema warehouse that metrics can be trusted against.

Approach

A staged extract → transform → load → data-quality → marts pipeline. Transform standardises dates and ISO-2 countries, dedupes, and converts amounts to EUR via an FX table; the load is idempotent (drop/recreate). A pytest suite enforces primary-key uniqueness, not-null, referential integrity and accepted-value checks, and exits non-zero on failure so it can gate CI.
